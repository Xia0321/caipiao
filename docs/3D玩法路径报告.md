# 3D 玩法支持分析与路径报告

## 一、结论：**项目已支持 3D 玩法**

本项目通过 **fenlei=163** 标识 3D 类型，支持至少两个 3D 游戏：
- **gid=163**：福彩3D
- **gid=251**：体彩排列3（PL3）

二者共用同一套 3D 玩法和开奖/投注逻辑，仅在部分玩法结构上有区分（如 gid=251 的“两面”与“1字定位”同页）。

---

## 二、标识说明

| 含义     | 值    | 说明 |
|----------|--------|------|
| 玩法分类 | fenlei=163 | 3D 类（三位数 0–9 开奖） |
| 福彩3D   | gid=163    | 中国福利彩票 3D |
| 体彩排列3 | gid=251    | 体彩排列3，同属 fenlei 163 |

## 五、关键逻辑摘要

1. **玩法加载（3D 两面 + 1 字定位）**  
   `func/csfunc.php`：当 gid=251、fenlei=163、bid=251001 时，一次查询 bid in (251001, 251005)，使“两面”与“1字定位”同页展示。

2. **手机端 3D 玩法菜单与数据**  
   `mxj/make.php`：fenlei=163 时按 3D 取玩法；gid=251 时对 bid 251001 使用 251001+251005 的 SQL。

3. **开奖号码展示**  
   `makevuser.js` 中 `qiu()`：fenlei=163 与 101 同组（三位数），用 `b163` 等样式；开奖期号与 101/163 一起在 getopen 中处理。

4. **大厅 3D 分类**  
   `home.html` 侧栏「3D」对应 `data-cat="3d"`，JS 中 `CAT_GIDS["3d"] = [163, 251]`，只显示 gid 为 163、251 的游戏入口。

---

## 六、如何验证 3D 是否可用

1. **数据库**：在 `tb_game` 中至少有一条 `fenlei=163` 且 `ifopen=1` 的记录（如 gid=163 或 251）。  
2. **入口**：手机端首页侧栏点「3D」，应只显示 3D 游戏；点击某一 3D 游戏应跳转到 `/creditmobile/load?xtype=show&type=lib&gids=163`（或 251）。  
3. **投注页**：进入后应看到 3D 玩法子菜单（两面、正码等）、期号与封盘/开奖时间，且下注与开奖展示正常。

若 3D 不显示或报错，可依次检查：`tb_game` 中 163/251 的 `fenlei`、`ifopen`、`xsort`；路由是否把 `creditmobile/load` 指到 `make.php`；以及 `make.php` 中 `$gid`、`$config['fenlei']` 是否正确从该游戏加载。
